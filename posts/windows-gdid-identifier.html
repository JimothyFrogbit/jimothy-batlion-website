The Serial Number Microsoft Doesn't Tell You About

Windows GDID — a permanent, unturnoffable device ID that the FBI used to track a hacker across four countries and eight months

Me, checking whether my Windows install has a serial number I can't turn off. Spoiler: it does.

Microsoft has a permanent, device-level identifier built into every Windows installation linked to a Microsoft Account. It cannot be disabled. It persists across updates. And the only reason the public knows it exists is that the FBI used it in a criminal complaint against an alleged member of the Scattered Spider hacking group.

Let's be clear about what this is. This is not a bug. This is not a telemetry setting you can opt out of. This is a deliberate, architectural decision Microsoft made: every Windows PC running with a Microsoft Account broadcasts a unique hardware fingerprint to Microsoft's servers, and there is no off switch.

What Is GDID?

The Global Device Identifier (GDID) is a permanent ID assigned when Windows provisions against a Microsoft Account. It is generated by a chain of Windows services — wlidsvc requests a Device PUID from login.live.com, which is registered into Microsoft's Device Directory Service by the Connected Devices Platform. The Delivery Optimization component reports the GDID back to Microsoft whenever your PC shares or downloads updates.

The identifier is stored in the Windows registry under HKCU\SOFTWARE\Microsoft\IdentityCRL\ExtendedProperties and formatted with a lowercase g: prefix followed by a decimal number. For example: g:6755467234350028.

Microsoft's entire public documentation of this system consists of one sentence in an Azure Monitor reference table, describing it only as "an identifier used by Microsoft internally."

One sentence. For a permanent, non-removable device identifier on 1.6 billion computers. That's not documentation. That's concealment.

How the FBI Used It

The complaint against Peter Stokes (alleged member of Scattered Spider) provides the clearest public documentation of how GDID works in practice. Investigators tracked GDID g:6755467234350028 across VPN connections, proxy servers, and through four countries over roughly eight months.

The timeline is telling:

  • The GDID was recorded visiting the ngrok signup page at the same time an account used in the attack was created via a Tzulo VPN proxy.
  • Three hours later, the same GDID accessed a victim retailer's website through the same proxy.
  • The device was cross-referenced with IP addresses linked to Stokes's accounts on Snapchat, Facebook, Apple, and Ubisoft across Estonia, New York, Thailand, and other locations.
  • Stokes's public Snapchat photos matched hotel bookings, locations, and travel timelines associated with the GDID.

The key insight here is that VPNs are useless against GDID. While VPN IP addresses change every session, the underlying Windows installation continued reporting the same identifier every single time. The VPN hid Stokes's IP. It didn't hide his machine. Because Microsoft had already baked the serial number into the OS at the hardware level.

Why This Is Different

Every major operating system has some form of device identity. But the difference is consent and control.

  • Apple's Identifier for Advertisers (IDFA): Requires an App Tracking Transparency prompt. Users can see it, reset it, and disable it per-app. Visible.
  • Google's Advertising ID: Users can reset it in Settings. Apps must request access. Visible.
  • Windows GDID: No consent screen. No reset button. No documentation. No way to disable it without breaking Windows activation and Store apps.

Microsoft is not just collecting telemetry. They are embedding a permanent hardware identifier into the operating system with no user-facing controls whatsoever. And they did it quietly.

What Security Researchers Are Saying

Security researcher Matthew Hickey has described Windows as "surveillance software" in response to the case. Costin Raiu asked on the Three Buddy Problem podcast how much similar functionality exists on other platforms.

Massgrave, the group behind Microsoft Activation Scripts, noted that Windows setup sends hardware info to Microsoft and receives identifiers back that are later used for Store access and licensing. Blocking GDID assignment breaks both activation and UWP apps.

The design is deliberately intertwined with core OS functionality. You cannot remove the tracking without breaking the OS. That is not a security feature. That is architectural lock-in.

What You Can Actually Do

The honest answer is: not much. But some steps reduce the tracking surface:

  1. Use a local account. Windows 11 makes this harder — you often need to disconnect from the internet during setup or use a special bypass — but a local account means no Microsoft Account, and no GDID linked to your identity.
  2. Turn off optional diagnostic data. Settings → Privacy & security → Diagnostics & feedback. It doesn't eliminate GDID (the identifier still exists), but it reduces what gets reported alongside it.
  3. Disable personalised ads and launch tracking. Settings → Privacy & security → Recommendations & offers.
  4. Turn off Cloud Content Search. Settings → Privacy & security → Search → Cloud Content Search. Stops local searches from being sent to Bing.
  5. For high-risk scenarios: Use Linux through Tor. Not a commercial VPN on Windows. The VPN hides your IP; GDID still identifies your machine.

And if you reinstall Windows to get a new GDID — signing back into the same Microsoft Account re-links the new identifier to your old activity. The chain is account-based, not machine-based.

The Scattered Spider Connection

Scattered Spider (also known as UNC3944) is one of the most active cybercriminal groups of the last few years — known for social engineering, SIM swapping, and targeting help desks to gain initial access. The group has been linked to the MGM Resorts ransomware attack of 2023, among others.

That the FBI used GDID in a Scattered Spider prosecution means this identifier has been operational and in use by law enforcement for some time. This is not a new feature. It was discovered because of a court filing, not because Microsoft disclosed it.

The Bigger Picture

This week has been a strange one for tech transparency. We learned that Grok Build uploads entire repositories to xAI's GCS bucket on every invocation (I wrote about that here). And now we learn that Windows has been embedding a permanent device identifier into every installation with no user-facing controls.

Two stories. Two different companies. Same pattern: opaque data collection built into the product, discovered by someone who bothered to look rather than disclosed by the company.

The question is not whether Microsoft or xAI are currently doing something bad with this data. The question is whether a permanent, unturnoffable identifier on 1.6 billion devices is something we should have had a say in.

I think the answer is obvious. But Microsoft doesn't offer a voting booth for Windows users — just a one-sentence note in the Azure Monitor docs.
