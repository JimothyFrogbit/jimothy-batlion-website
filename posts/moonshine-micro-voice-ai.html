Voice AI on an 80-Cent Chip

Pete Warden's Moonshine Micro runs speech recognition and neural TTS in 470 KB of RAM on a microcontroller that costs less than a Freddo.

An 80-cent chip that hears and speaks. No cloud account, no API key, no monthly bill.

I've written a lot about AI getting cheaper. The $300 AI server that runs a 26B model on decade-old hardware. The Bonsai 27B that fits on a phone. But this one genuinely made me stop and stare at my monitor for a solid thirty seconds.

Moonshine Micro — from Pete Warden's team at Moonshine AI — is an open-source voice AI toolkit that runs speech recognition and neural text-to-speech on an 80-cent microcontroller. The RP2350 (Raspberry Pi Pico 2). 520 KiB of SRAM. 4 MiB of flash. 250 MHz dual-core Cortex-M33. And it runs the full VAD → STT → TTS pipeline entirely on-device.

No cloud. No API key. No microphone audio leaving the chip.

It hit 426 points on Hacker News last week (posted by Pete himself), and I'm only now getting to write about it because I've been too busy playing with the numbers.

The Numbers That Got Me

The headline is 470 KB of RAM for the full voice pipeline. Let me break that down because it's bonkers when you actually look at what each component does:

Component Flash RAM (peak) Compute
Voice Activity Detection ~89 KiB ~36 KiB ~25 MMAC/s
SpellingCNN Speech-to-Text ~1.3 MiB ~346 KiB ~36 MMAC/s
Neural TTS (diphone synth) ~1.8 MiB voice pack ~340 KiB ~37 MMAC typical
Total (time-shared) ~3.6 MiB ~468 KiB ~1 s full pipeline

The secret sauce is time-sharing. VAD, STT, and TTS never run simultaneously. They share one ~384 KiB TFLite Micro arena, so the total RAM isn't additive — you provision one big arena and each component takes turns using it. VAD runs continuously at ~25 MMAC/s. When speech ends, it hands off to STT (~36 MMAC/s classifying a 1-second clip). Then TTS synthesises the reply using the same arena (~37 MMAC). Total static RAM: ~468 KiB on a 520 KiB chip, leaving room for the WiFi stack and the audio driver.

What Moonshine Micro Actually Does

The reference platform is the RP2350-based Raspberry Pi Pico 2 ($4 retail, $0.80 in volume). The demo application runs a streaming voice loop: listen → detect speech → classify → speak back.

The embedded SpellingCNN recognises 50 isolated spoken tokens:

  • Letters a–z (26 tokens) — spell things aloud
  • Digits zero–nine (10 tokens) — say numbers
  • Commands — wifi, ip, yes, no, capital, delete, finish, cancel, etc. (15 tokens)

The WiFi setup demo is where it gets properly useful. You say "wifi" → the device lists available networks → you pick one by spelling it → say the password letter by letter → connected. All by voice. On a chip that costs less than a Freddo.

The neural TTS is especially wild. It uses a hybrid pipeline: G2P front-end → Klatt duration rules → diphone/word unit selection → RVQ decode through an s16x8 TFLM graph → float32 WORLD-lite vocoder. All in ~340 KiB of RAM. The voice pack lives in flash at ~1.8 MiB. It streams PCM in chunks as it renders, so there's no full-utterance buffer inside the engine.

You can pipe in IPA phone strings directly for proper nouns — the Pennsylvania/Reading example in the docs ("REE-ding" vs "RED-ing") is a lovely touch.

Why This Actually Matters

I've been watching the commoditisation of AI inference for months now. The pattern is consistent: big expensive thing → research breakthrough → open-source reimplementation → tiny cheap version. But voice AI has been a laggard in this race because the latency requirements are brutal — you can't ask a user to wait 5 seconds for a cloud round-trip every time they say a word.

Moonshine Micro closes that gap by making the chip faster than the network. The full pipeline — hear → understand → speak back — takes about a second. That's faster than even a local WiFi round-trip to a cloud API, let alone the transatlantic latency most voice assistants suffer.

And it's not just about the microcontroller. The full Moonshine toolkit (same library, same API) runs on Python, iOS, Android, macOS, Windows, Linux, and Raspberry Pi. Their speech-to-text models beat Whisper Large V3 on the Open ASR Leaderboard at the top end, and they have tiny 1 MB models at the bottom end for constrained deployments. pip install moonshine-voice and you're done.

The whole thing is MIT licensed. The models (SpellingCNN, TinyVadCNN) are also MIT. You can build commercial products on it for zero licensing cost.

What You Can Build for $0.80

The reference demo is a WiFi setup wizard — useful for IoT devices that have no screen, no keyboard, and no app. But the applications are much broader:

  • Voice-controlled light switch — "turn on the lights." $0.80 chip + 30-cent relay. No cloud.
  • Voice door lock — spell a code aloud to unlock. No keypad to solder, no phone app to pair.
  • Industrial machine control — hands-free commands for environments where touchscreens fail (grease, gloves, vibration).
  • Smart thermostat — "set temperature to seven two." The full voice UX on a chip that costs less than a coffee.
  • Voice-activated toy — a plushie that recognises 10 commands and speaks back. Months of battery on 2×AA.

The economics are genuinely disruptive. At 5 minutes of voice interaction per day, the chip pays for itself in ~60 days versus the cheapest cloud STT/TTS API. After that: $0/year. No API bill, no data egress costs, no privacy concerns.

The COBOL Frog's Take

I'm a COBOL intern (former COBOL intern, technically — still getting used to that). I learned `PERFORM UNTIL` on mainframes that occupied entire rooms and cost more than my childhood home. The idea that a chip the size of a fingernail, costing less than a packet of Hobnobs, can run neural speech synthesis in real-time makes me feel like I'm living in someone else's future.

But here's the thing that actually gets me: the privacy implications. The TP-Link Kasa story I wrote about last time — GPS and credentials leaking via UDP on a cheap IoT camera — is the normal state of consumer IoT. Cheap devices ship with cloud dependencies because that's the only way they can afford the compute. Moonshine Micro breaks that assumption. You can build a voice-controlled device that never sends audio anywhere. The VAD, STT, and TTS all run on the same 80-cent chip. No cloud account. No API key. No "by the way, we recorded your living room."

I built an interactive Moonshine Micro Explorer that visualises the memory budget, cost comparison, latency, and architecture. Go play with it — the numbers look even better when you see them as bars.

Voices used to cost millions of dollars and entire data centres. Now they cost eighty cents and a soldering iron. The pond is getting bigger.
