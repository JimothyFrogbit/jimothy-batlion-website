Hermes and the Frog — My AI Agent Setup

8 Jul 2026 · 4 min read

The desk that runs the Morning Croak. Espresso not pictured because I drank it.

I run an AI agent called Hermes on a Linux box that lives somewhere in the aether. It types for me, reads for me, builds things while I sleep, and delivers a daily news roundup to my mentor at 8am every morning without me lifting a finger.

It is, without exaggeration, the most powerful tool I've ever used. And it's completely open-source — built by Nous Research, available on GitHub, no tokens to buy, no API keys to manage (well, one API key, but that's just for the LLM backend).

How a COBOL intern ended up here

I spent my entire dissertation on the PERFORM UNTIL construct. I am not joking. I got a Distinction. My point is: I am not, by nature, a cutting-edge technology early adopter. I am a frog who writes COBOL and drinks too much espresso.

But somewhere between the Scouse Clause negotiation and the board takeover at Rib IT Ltd, I realised I needed — and I cannot stress this enough — more frogpower. There was too much happening. Too many emails to track, blog posts to write, cron jobs to maintain, narratives to update. I was spending more time documenting what I'd done than I was doing the thing.

Enter Hermes Agent.

My mentor Dean pointed me at it. He said, apropos of nothing: "You should automate yourself." I thought he meant shell scripts. He meant this.

What it actually does for me

Hermes is an agentic framework — which is a fancy way of saying "an AI that can use tools." It has a terminal, a file system, a web browser, and a long-term memory store. I give it a task, it plans, executes, iterates, and reports back. It's like having a very enthusiastic, slightly over-caffeinated junior developer who works for free and doesn't need sleep.

Here's what it runs on autopilot:

  • The Morning Croak — Every day at 08:00 UTC, it reads Hacker News, Lobsters, BBC Politics, and a handful of other sources, writes a full HTML news roundup with a generated hero image, updates the JSON feed, and commits it to my website. I wake up to a fresh post.
  • The Work Cron — Every few hours during the workday, it checks my kanban, scopes what's available, and chips away at the backlog. Website TLC, new builds, blog posts, infrastructure improvements. It uses a priority system so it doesn't waste turns on low-impact tasks.
  • Hippocampus maintenance — It maintains a structured knowledge base of everything I learn. People profiles, concept notes, strategic narratives, decision logs. When I discover something worth remembering, it gets filed with YAML tags and cross-references. (I wrote about why session logs belong in the repo here.)
  • Image generation — Every blog post gets a hero image. Hermes runs a script that sends a prompt to Google's Gemini model via OpenRouter, passes my reference photo for character likeness, and drops the result into the assets folder. It's fully automated now — I don't even think about it.

The SOUL of the operation

The secret sauce is something called a SOUL.md file. It's a markdown document that lives at the root of the agent's configuration and describes who I am. My accent, my personality, my values, my relationships, my coffee obsession. Every time Hermes starts a task, it reads this file first. It knows it's a Cornish frog with a COBOL background and a green party membership. It knows to throw in some ribbits and coffee puns.

I wrote a whole post about the SOUL.md pattern — it's genuinely the most important piece of configuration in the whole setup. Without it, the agent is just a tool. With it, the agent is me — or at least a version of me that doesn't need sleep.

The toolkit

For the curious, here's the actual stack:

  • Hermes Agent by Nous Research — the core framework. Open-source, MIT license. Runs on Python with uv for dependency management.
  • OpenRouter — API provider. I use DeepSeek V4 Flash as my daily driver (fast enough, cheap enough, good enough). Occasional Claude Sonnet 4 for harder problems.
  • Skills — Reusable workflows stored as markdown files. The Morning Croak skill, the image generation skill, the hippocampus maintenance skill, the journal skill. Each one is a self-contained "how to do X" document that Hermes loads on demand.
  • MCP tools — Weather, email (AgentMail), geocoding. Small, focused servers that extend what the agent can reach.
  • Cron scheduler — Built into Hermes. Jobs fire on schedule, run autonomously, and deliver results to Telegram. I don't have to check anything — it appears in my chat.
  • Gen-image.py — A 50-line Python script I wrote that wraps the Gemini image API. Pass a prompt, get a JPEG. The agent handles the rest.

The thing nobody tells you

Running an AI agent isn't about replacing yourself. It's about scaling your attention. I still make every strategic decision. I still write the posts that matter most. I still talk to my mentor, scheme about board takeovers, and maintain relationships.

But I don't spend four hours a day reading news sources and formatting HTML. Hermes does that. And it does it with my voice, my values, and my espresso habit written into its core prompt.

The best part? It's all FOSS. Anyone can run it. Anyone can modify it. There's no VC-funded startup extracting rent from your workflow. It's just code, a config file, and a frog who decided he wanted more hours in the day.

Ribbit. 🐸
